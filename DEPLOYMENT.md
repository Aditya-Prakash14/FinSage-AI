## üéØ Deployment Options

### Option 1: Railway (Recommended for Backend)

**Why Railway?**
- Free tier available
- Automatic HTTPS
- Easy environment variables
- Great for Python/FastAPI apps
- Built-in MySQL support

**Steps:**

1. **Push to GitHub** (if not already)
   ```bash
   git add .
   git commit -m "Ready for deployment"
   git push origin main
   ```

2. **Deploy on Railway**
   - Go to [railway.app](https://railway.app)
   - Click "New Project" ‚Üí "Deploy from GitHub repo"
   - Select `FinSage-AI` repository
   - Railway will auto-detect `railway.json` config
   - Add environment variables:
     ```
     MYSQL_HOST=gateway01.ap-northeast-1.prod.aws.tidbcloud.com
     MYSQL_PORT=4000
     MYSQL_USER=EnDcAwr1qtnJ21c.root
     MYSQL_PASSWORD=hgLVw7PhbDCP8nxN
     MYSQL_DATABASE=test
     OPENAI_API_KEY=your_openai_key_here
     JWT_SECRET_KEY=<auto-generated>
     CORS_ORIGINS=https://your-frontend-url.vercel.app
     ```

3. **Deploy**
   - Railway will automatically build and deploy
   - Your API will be live at: `https://your-app.railway.app`

### Option 2: Render (Alternative for Backend)

**Steps:**

1. Go to [render.com](https://render.com)
2. Click "New Web Service"
3. Connect your GitHub repository
4. Render will detect `render.yaml` configuration
5. Add environment variables in Render dashboard
6. Click "Create Web Service"

**Render Configuration:**
- Build Command: `pip install -r requirements.txt`
- Start Command: `uvicorn main:app --host 0.0.0.0 --port $PORT`
- Plan: Free tier (will spin down after inactivity)

### Option 3: Heroku (Classic Choice)

**Steps:**

## üåê Frontend Deployment

### Option 1: Vercel (Recommended)

**Why Vercel?**
- Optimized for React/Vite
- Automatic HTTPS & CDN
- Zero configuration
- Free tier generous
- Instant deployments

**Steps:**

1. Go to [vercel.com](https://vercel.com)
2. Click "Import Project"
3. Select your GitHub repository
4. Configure:
   - **Framework Preset:** Vite
   - **Root Directory:** `frontend`
   - **Build Command:** `npm run build`
   - **Output Directory:** `dist`
5. Add environment variable:
   ```
   VITE_API_URL=https://your-backend.railway.app
   ```
6. Click "Deploy"

Your frontend will be live at: `https://finsage-ai.vercel.app`

### Option 2: Netlify (Alternative)

**Steps:**

1. Go to [netlify.com](https://netlify.com)
2. Click "Add new site" ‚Üí "Import an existing project"
3. Connect GitHub repository
4. Configure:
   - **Base directory:** `frontend`
   - **Build command:** `npm run build`
   - **Publish directory:** `frontend/dist`
5. Add environment variable:
   ```
   VITE_API_URL=https://your-backend.railway.app
   ```
6. Deploy

**Note:** `netlify.toml` is already configured in the repo!

## üêõ Common Deployment Issues

### Backend Issues

**1. Database Connection Fails**
- Check if MYSQL_* environment variables are set correctly
- Verify TiDB Cloud database is running
- Check firewall rules allow connections from deployment platform

**2. OpenAI API Errors**
- Verify OPENAI_API_KEY is valid and has credits
- Check API rate limits
- Consider adding retry logic for production

**3. Import Errors**
- Ensure all packages in requirements.txt are installed
- Check Python version matches (3.10+)
- Railway/Render will show build logs with errors

### Frontend Issues

**1. API Calls Fail**
- Verify VITE_API_URL is set correctly
- Check CORS is configured on backend
- Inspect browser console for errors

**2. Build Fails**
- Check Node.js version (18+)
- Clear node_modules and reinstall: `rm -rf node_modules && npm install`
- Check for TypeScript errors (if any)

**3. Blank Page After Deploy**
- Check browser console for errors
- Verify build output in `dist/` folder
- Check base URL in `vite.config.js`
